{% comment %} Override of unfold/helpers/app_list_badge.html. Why: Unfold's stock template always renders the badge when ``item.badge`` (the config-time string like ``"admin.badges.x"``) is truthy. When the callback resolves to ``None`` or ``0`` (no rows to flag), the Django lazy proxy's ``__str__`` returns ``"None"`` / ``"0"``, which the badge ``uppercase`` CSS class then renders as literally ``NONE`` / ``0`` — a confusing chip on otherwise-empty nav items. Fix: resolve the callback into a local ``count`` variable up-front and only render the span when ``count`` is a non-empty value. ``stringformat:"s"`` forces lazy evaluation so the ``{% if %}`` gate sees the actual result, not the always-truthy proxy object. {% endcomment %} {% if item.badge %} {% if item.badge_callback %} {% with count=item.badge_callback|stringformat:"s" %} {% if count and count != "None" and count != "0" %} {{ count }} {% endif %} {% endwith %} {% else %} {{ item.badge }} {% endif %} {% endif %}
